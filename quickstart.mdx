Get your first homelab running in under 5 minutes using the managed kombify Sphere platform.

## Prerequisites

<CardGroup cols={2}>
  <Card title="A server" icon="server">
    Any Linux server (VPS, old PC, Raspberry Pi)
  </Card>
  <Card title="SSH access" icon="terminal">
    SSH access to your server
  </Card>
</CardGroup>

## Step 1: Create an account

<Steps>
  <Step title="Sign up">
    Go to [app.kombisphere.io](https://app.kombisphere.io) and create a free account.
  </Step>

  <Step title="Verify email">
    Check your email and verify your account.
  </Step>

  <Step title="Access dashboard">
    You'll be redirected to the kombify Sphere dashboard.
  </Step>
</Steps>

## Step 2: Launch kombify Stack

<Steps>
  <Step title="Open tool launcher">
    Click **"Launch Stack"** in the dashboard.
  </Step>

  <Step title="Choose deployment">
    Select **"Managed"** to use Sphere's hosted Stack instance.
    
    <Note>
      You can also choose **"Self-hosted"** to run Stack on your own infrastructure.
    </Note>
  </Step>

  <Step title="Access Stack">
    You'll be automatically logged in via SSO.
  </Step>
</Steps>

## Step 3: Create your first stack

<Steps>
  <Step title="Start wizard">
    Click **"New Homelab"** and select **"Easy Wizard"**.
  </Step>

  <Step title="Answer questions">
    The wizard will ask:
    - What do you want to build? (e.g., "Photo Cloud")
    - How many users? (e.g., "4 (Family)")
    - Where should it run? (e.g., "Local Server")
    - Do you have a domain? (optional)
  </Step>

  <Step title="Review configuration">
    The wizard generates a `kombination.yaml` file:
    
    ```yaml
    stackkit: base-homelab
    version: "1.0.0"
    
    meta:
      name: "My Photo Cloud"
    
    nodes:
      - name: main-server
        type: local
        connection:
          host: 192.168.1.100
    
    services:
      - traefik
      - uptime-kuma
      - immich
    ```
  </Step>
</Steps>

## Step 4: Simulate (optional)

<Steps>
  <Step title="Start simulation">
    Click **"Simulate"** to test your config in a virtual environment.
  </Step>

  <Step title="Test services">
    Access the simulated services:
    - Traefik: `http://localhost:8080`
    - Immich: `http://localhost:2283`
    - Uptime Kuma: `http://localhost:3001`
  </Step>

  <Step title="SSH into simulation">
    ```bash
    ssh -p 2222 root@localhost
    # Password: kombisim
    ```
  </Step>

  <Step title="Stop simulation">
    When satisfied, click **"Stop Simulation"**.
  </Step>
</Steps>

## Step 5: Deploy to real hardware

<Steps>
  <Step title="Install agent">
    Copy the agent installation command from the dashboard:
    
    ```bash
    curl -sSL https://install.kombify.io/agent | sudo sh -s -- \
      --token YOUR_UNIQUE_TOKEN \
      --server https://api.kombisphere.io
    ```
    
    Run this on your server.
  </Step>

  <Step title="Wait for connection">
    The dashboard will show **"Agent Connected"** when ready.
  </Step>

  <Step title="Deploy">
    Click **"Apply"** to start the deployment.
    
    You'll see real-time progress:
    ```
    ✅ Validating configuration...
    ✅ Connecting to node: main-server
    ✅ Installing Docker services...
       ├─ traefik: running
       ├─ uptime-kuma: running
       └─ immich: running
    ✅ Deployment complete!
    ```
  </Step>
</Steps>

## Step 6: Access your services

Your services are now running! Access them at:

| Service | URL | Purpose |
|---------|-----|---------|
| **Traefik Dashboard** | `http://your-server:8080` | Reverse proxy status |
| **Uptime Kuma** | `http://your-server:3001` | Service monitoring |
| **Immich** | `http://your-server:2283` | Photo management |

<Note>
  If you configured a domain, services will be available at `service.yourdomain.com` with automatic SSL.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent not connecting">
    Check the agent status on your server:
    
    ```bash
    sudo systemctl status kombify-agent
    sudo journalctl -u kombify-agent -f
    ```
  </Accordion>

  <Accordion title="Container not starting">
    Check Docker logs:
    
    ```bash
    docker logs traefik
    docker logs immich
    ```
  </Accordion>

  <Accordion title="Can't access services">
    - Check firewall rules: `sudo ufw status`
    - Verify ports are open: `sudo netstat -tlnp | grep -E "80|443|2283"`
    - Check Docker network: `docker network ls`
  </Accordion>
</AccordionGroup>
