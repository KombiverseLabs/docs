# Templates API

Templates are pre-configured simulation blueprints for common homelab scenarios. Use them to quickly spin up realistic test environments.

## Apply template

Create a simulation from a template:

```http
POST /api/v1/templates/{template_id}/apply
```

### Request body

```json
{
  "simulation_name": "my-homelab-test",
  "customizations": {
    "nodes": {
      "web-server": {
        "resources": {
          "memory_mb": 4096
        }
      }
    }
  },
  "start_immediately": true
}
```

### Response

```json
{
  "simulation_id": "sim_abc123",
  "name": "my-homelab-test",
  "status": "creating",
  "template_applied": "homelab-basic",
  "nodes": [
    {
      "id": "node_111",
      "name": "web-server",
      "status": "creating"
    },
    {
      "id": "node_222",
      "name": "database",
      "status": "creating"
    }
  ]
}
```

## Create custom template

Save a simulation as a reusable template:

```http
POST /api/v1/templates/create
```

### Request body

```json
{
  "name": "my-custom-template",
  "description": "Custom template for my specific use case",
  "category": "custom",
  "source_simulation_id": "sim_abc123"
}
```

### Response

```json
{
  "id": "custom-my-custom-template",
  "name": "my-custom-template",
  "description": "Custom template for my specific use case",
  "category": "custom",
  "created_at": "2026-02-03T10:00:00Z"
}
```

## Built-in templates reference

### single-vps

Simulates a single VPS server:

| Property | Value |
|----------|-------|
| **Nodes** | 1 |
| **OS** | Ubuntu 24.04 |
| **CPU** | 1 vCPU |
| **Memory** | 1 GB |
| **Use case** | Testing single-server deployments |

### single-local

Simulates a local server:

| Property | Value |
|----------|-------|
| **Nodes** | 1 |
| **OS** | Ubuntu 24.04 |
| **CPU** | 2 vCPU |
| **Memory** | 2 GB |
| **Use case** | Testing local homelab setups |

### hybrid-homelab

Simulates VPS + local server combination:

| Property | Value |
|----------|-------|
| **Nodes** | 2 (1 VPS, 1 local) |
| **OS** | Ubuntu 24.04 |
| **Use case** | Testing hybrid cloud-local setups |

Nodes:
- `vps-entry` — Public-facing VPS with Traefik
- `local-server` — Local server for applications

### homelab-basic

Basic homelab with web and database:

| Property | Value |
|----------|-------|
| **Nodes** | 2 |
| **OS** | Ubuntu 24.04 |
| **Use case** | Testing standard homelab deployments |

Nodes:
- `web-server` — Nginx, applications
- `database` — PostgreSQL, Redis

### homelab-advanced

Full-featured homelab:

| Property | Value |
|----------|-------|
| **Nodes** | 5 |
| **OS** | Ubuntu 24.04 |
| **Use case** | Testing complex deployments |

Nodes:
- `proxy` — Traefik reverse proxy
- `auth` — Authelia authentication
- `apps` — Application server
- `database` — PostgreSQL
- `storage` — MinIO object storage

### ha-setup

High-availability test setup:

| Property | Value |
|----------|-------|
| **Nodes** | 5 |
| **OS** | Ubuntu 24.04 |
| **Use case** | Testing HA configurations |

Nodes:
- `lb-1`, `lb-2` — HAProxy load balancers
- `web-1`, `web-2`, `web-3` — Backend servers

### dev-environment

Development and testing environment:

| Property | Value |
|----------|-------|
| **Nodes** | 2 |
| **OS** | Ubuntu 24.04 |
| **Use case** | Local development testing |

Nodes:
- `dev-server` — Development server
- `db-server` — Database server

## Template categories

| Category | Description |
|----------|-------------|
| `vps` | Single VPS simulations |
| `local` | Local server simulations |
| `homelab` | Full homelab setups |
| `development` | Development environments |
| `custom` | User-created templates |
