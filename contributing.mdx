### Fork and clone

<Steps>
  <Step title="Fork the repository">
    Visit the repository on GitHub and click **Fork**:
    - [kombify Stack](https://github.com/soulcreek/KombiStack)
    - [kombify Sim](https://github.com/soulcreek/KombiSim)
    - [StackKits](https://github.com/soulcreek/StackKits)
  </Step>

  <Step title="Clone your fork">
    ```bash
    git clone https://github.com/YOUR-USERNAME/KombiStack.git
    cd KombiStack
    ```
  </Step>

  <Step title="Add upstream remote">
    ```bash
    git remote add upstream https://github.com/soulcreek/KombiStack.git
    ```
  </Step>
</Steps>

## Making changes

### Create a branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
```

### Make your changes

Follow our coding standards:

<Tabs>
  <Tab title="Go">
    ```go
    // Use gofmt
    gofmt -w .
    
    // Run linter
    golangci-lint run
    
    // Run tests
    go test ./...
    ```
  </Tab>

  <Tab title="TypeScript">
    ```bash
    # Format code
    npm run format
    
    # Lint
    npm run lint
    
    # Type check
    npm run check
    ```
  </Tab>

  <Tab title="CUE">
    ```bash
    # Format
    cue fmt
    
    # Validate
    cue vet
    ```
  </Tab>
</Tabs>

### Write tests

All new features should include tests:

```go
// Example test
func TestUnifierResolve(t *testing.T) {
    config := &Config{
        Services: []string{"traefik"},
    }
    
    result, err := Unifier.Resolve(config)
    assert.NoError(t, err)
    assert.NotNil(t, result)
}
```

### Commit your changes

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```bash
# Format: <type>(<scope>): <description>

git commit -m "feat(stack): add support for custom networks"
git commit -m "fix(sim): resolve SSH connection timeout"
git commit -m "docs: update Proxmox integration guide"
```

**Types:**
- `feat` â€” New feature
- `fix` â€” Bug fix
- `docs` â€” Documentation changes
- `style` â€” Code style changes (formatting)
- `refactor` â€” Code refactoring
- `test` â€” Adding tests
- `chore` â€” Maintenance tasks

## Submitting a pull request

<Steps>
  <Step title="Push your branch">
    ```bash
    git push origin feature/your-feature-name
    ```
  </Step>

  <Step title="Create pull request">
    Go to GitHub and click **New Pull Request**
  </Step>

  <Step title="Fill out template">
    Provide:
    - Clear description of changes
    - Related issue number (if applicable)
    - Screenshots (for UI changes)
    - Testing steps
  </Step>

  <Step title="Wait for review">
    Maintainers will review your PR and may request changes
  </Step>
</Steps>

### Pull request checklist

- [ ] Code follows project style guidelines
- [ ] Tests pass locally
- [ ] New tests added for new features
- [ ] Documentation updated
- [ ] Commit messages follow Conventional Commits
- [ ] No merge conflicts
