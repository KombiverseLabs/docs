## SSO Configuration

### Supported providers

- Okta
- Azure Active Directory
- Google Workspace
- OneLogin
- Ping Identity
- Any SAML 2.0 / OIDC provider

### OIDC setup

<Steps>
  <Step title="Get kombify Cloud OIDC details">
    In kombify Cloud: **Settings** → **Security** → **SSO**

    Copy:
    - Redirect URI: `https://app.kombify.io/auth/callback`
    - Client ID: (provided)
  </Step>
  <Step title="Configure your IdP">
    In your identity provider:
    - Add new OIDC application
    - Set redirect URI
    - Configure scopes: `openid`, `profile`, `email`
    - Copy Client Secret
  </Step>
  <Step title="Complete kombify Cloud setup">
    Enter in kombify Cloud:
    - Issuer URL
    - Client ID
    - Client Secret
  </Step>
  <Step title="Configure role mapping">
    Map IdP groups to kombify Cloud roles:
    - `kombify-admins` → Admin
    - `kombify-operators` → Operator
    - `kombify-viewers` → Viewer
  </Step>
  <Step title="Enable SSO">
    Toggle **Enable SSO**
    
    Optionally: **Enforce SSO** (disable password login)
  </Step>
</Steps>

### SAML setup

<Accordion title="SAML configuration details">
**kombify Cloud SAML metadata:**
- Entity ID: `https://app.kombify.io/saml/metadata`
- ACS URL: `https://app.kombify.io/saml/acs`
- SLO URL: `https://app.kombify.io/saml/slo`

**Required attributes:**
- `email` (required)
- `firstName` (optional)
- `lastName` (optional)
- `groups` (for role mapping)
</Accordion>

### SCIM provisioning

Automatically sync users from your IdP:

1. Enable SCIM in **Settings** → **Security** → **SCIM**
2. Copy SCIM endpoint and token
3. Configure SCIM in your IdP
4. Map groups to kombify Cloud roles

**Supported operations:**
- Create user
- Update user
- Deactivate user
- Group membership sync

## Self-hosted deployment

Run kombify Cloud in your own infrastructure.

### Deployment options

| Option | Description |
|--------|-------------|
| **Docker Compose** | Standard deployment via Docker Compose |
| **Render** | Managed deployment via Render PaaS |

### Requirements

**Minimum:**
- 4 CPU cores
- 16 GB RAM
- 100 GB SSD storage
- Docker 24+
- Docker Compose v2
- PostgreSQL 17

### Installation

<Steps>
  <Step title="Get license">
    Contact sales for Enterprise license key
  </Step>
  <Step title="Download package">
    ```bash
    docker pull ghcr.io/kombiverse/kombify-cloud:enterprise
    ```
  </Step>
  <Step title="Configure">
    Edit `docker-compose.yaml`:
    ```yaml
    license_key: "your-license-key"
    domain: "cloud.yourcompany.com"
    database:
      host: "postgres.internal"
      name: "kombify"
    ```
  </Step>
  <Step title="Deploy">
    ```bash
    docker compose up -d
    ```
  </Step>
  <Step title="Initialize">
    Access `https://cloud.yourcompany.com/setup` to complete initial configuration.
  </Step>
</Steps>

### Updates

Self-hosted instances receive updates via:
- **Automatic** — Enabled by default, applies during maintenance window
- **Manual** — Download and apply updates yourself

```bash
# Pull latest images
docker compose pull

# Apply update
docker compose up -d
```
