# kombify Sim

kombify Sim is an infrastructure simulation engine that lets you test homelab configurations before deploying to real hardware using lightweight Docker containers.

<Card title="What is kombify Sim?" icon="circle-info">
  kombify Sim creates Docker-based "virtual servers" that simulate your entire homelab environment. You can SSH into them, test configurations, and validate your setup before touching real hardware.
</Card>

## Key features

<CardGroup cols={2}>
  <Card title="Docker simulation" icon="docker">

  </Card>
  <Card title="Real SSH access" icon="terminal">

## How it works

```
┌─────────────────────────────────────────────────────────────┐
│                        kombify Sim                           │
│  ┌─────────────────┐         ┌─────────────────┐           │
│  │    Web UI       │────────▶│    REST API     │           │
│  │   (SvelteKit)   │         │      (Go)       │           │
│  │   Port: 5271    │         │   /api/v1/*     │           │
│  └─────────────────┘         └────────┬────────┘           │
│                                       │                     │
│                          ┌────────────▼────────────┐        │
│                          │   Node Engine           │        │
│                          │   (Docker Backend)      │        │
│                          └────────────┬────────────┘        │
│                                       │                     │
│                          ┌────────────▼────────────┐        │
│                          │   Docker Daemon         │        │
│                          │   (Containers as VMs)   │        │
│                          └─────────────────────────┘        │
└─────────────────────────────────────────────────────────────┘
```

## Built-in templates

| Template | Nodes | Description |
|----------|-------|-------------|
| `single-node` | 1 | Minimal single server |
| `homelab-basic` | 2 | Web server + Database |
| `homelab-advanced` | 5 | Full homelab stack |
| `hybrid-setup` | 5 | VPS + Local combined |
| `ha-setup` | 5 | Load balancer + HA |

## Service ports

| Service | Port | Protocol | Description |
|---------|------|----------|-------------|
| Backend API | 5270 | HTTP | REST API |
| Frontend | 5271 | HTTP | SvelteKit Dashboard |
| SSH (Nodes) | 2222-2322 | SSH | Container SSH access |

## Quick example

```bash
# Create a simulation
curl -X POST http://localhost:5270/api/v1/simulations \
  -H "Content-Type: application/json" \
  -d '{
    "name": "test-lab",
    "description": "Testing my homelab setup"
  }'

# Add a node
curl -X POST http://localhost:5270/api/v1/nodes \
  -H "Content-Type: application/json" \
  -d '{
    "name": "web-server",
    "simulation_id": "sim-uuid",
    "type": "vps",
    "os": "ubuntu-22.04"
  }'

# SSH into the node
ssh -p 2222 root@localhost
# Password: kombisim
```
