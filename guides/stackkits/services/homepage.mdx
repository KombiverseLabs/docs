Homepage is the optional secondary start page at `home.<domain>`. StackKits generates its service groups from the same service catalog as the Node Hub.

## Prerequisites

<Note>
  Homepage appears only when `enable_homepage` is enabled for the rollout.
</Note>

## Steps

<Steps>
  <Step title="Open Homepage">
    Open `https://home.<domain>` or use the Homepage link in the Node Hub.
  </Step>
  <Step title="Check service groups">
    Confirm that Platform and Applications match the services you enabled in the rollout.
  </Step>
  <Step title="Use Node Hub for setup">
    Return to `base.<domain>` for first-run steps and how-to links. Homepage is meant for daily launch, not recovery.
  </Step>
</Steps>

## Verify

- Homepage loads without a writable Docker socket mount.
- Service links open in a new tab.
- Disabled services do not appear.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Homepage is empty">
    Regenerate the StackKits output so Homepage receives the current service catalog.
  </Accordion>
</AccordionGroup>
