# StackKits

StackKits are **declarative infrastructure blueprints** that combine CUE validation with OpenTofu execution. They provide battle-tested configurations for common homelab scenarios.

## What is a StackKit?

A StackKit is a complete infrastructure template that includes:

- **CUE schemas** for type-safe validation
- **OpenTofu templates** for provisioning
- **Service definitions** for applications
- **OS variants** for different distributions
- **Default configurations** for quick starts

## Key features

<AccordionGroup>
  <Accordion title="3-layer architecture" icon="layer-group">
    StackKits use a strict 3-layer architecture:
    
    **Layer 1: Core** - Shared foundation (security, network, observability)
    
    **Layer 2: Platforms** - Container orchestration (Docker, Kubernetes)
    
    **Layer 3: StackKits** - Use-case specific configurations
  </Accordion>

  <Accordion title="CUE validation" icon="shield-check">
    Constraint-based validation catches errors before deployment:
    
    ```cue
    #BaseHomelabKit: {
        nodes: [...#Node]
        nodes: {
            // At least one main node required
            [_]: type: "main" | "worker"
        }
    }
    ```
  </Accordion>

  <Accordion title="Multi-OS support" icon="ubuntu">
    OS variants for different distributions:
    
    - Ubuntu 24.04, 22.04
    - Debian 12, 11
    - More coming soon
  </Accordion>

  <Accordion title="Deployment modes" icon="rocket">
    **Simple mode:** Single OpenTofu file
    
    **Advanced mode:** Terramate-orchestrated stacks
  </Accordion>
</AccordionGroup>
