The kombify Cloud dashboard is your central hub for managing all connected homelabs. Monitor status, deploy updates, and collaborate with your team from a single interface.

- **Connected homelabs** ‚Äî All stacks connected via kombify Agent

## Homelab cards

Each connected homelab displays as a card with:

| Element | Description |
|---------|-------------|
| **Name** | Stack name (e.g., "Production Homelab") |
| **Status** | online, degraded, offline |
| **Version** | Current StackKit version |
| **Nodes** | Number of nodes / running |
| **Last seen** | Agent heartbeat timestamp |
| **Quick actions** | Deploy, View, Settings |

## Filtering and search

Filter homelabs by:
- **Status** ‚Äî Online, Degraded, Offline
- **StackKit** ‚Äî base-homelab, dev-homelab, etc.
- **Tag** ‚Äî Custom labels (production, staging)
- **Owner** ‚Äî Team member

Search by name:
```
production main-server dokploy
```

## Quick actions

### Deploy update

1. Click **Deploy** on a homelab card
2. Select target version or "Latest"
3. Review changes
4. Click **Deploy Now** or **Schedule**

### View details

Click a homelab card to see:
- Full node list with health
- Service status
- Recent logs
- Configuration

### Manage settings

Access homelab settings to:
- Edit name and tags
- Configure notifications
- Set maintenance windows
- Manage access permissions

## Notifications

Configure alerts for:

<Accordion title="Health alerts">
Trigger when nodes become unhealthy:
- Node offline
- High CPU/memory
- Disk space low
- Service crashed
</Accordion>

<Accordion title="Deployment alerts">
Trigger on deployment events:
- Deployment started
- Deployment completed
- Deployment failed
- Rollback triggered
</Accordion>

<Accordion title="Security alerts">
Trigger on security events:
- Failed login attempts
- Certificate expiring
- Unauthorized access attempt
</Accordion>
