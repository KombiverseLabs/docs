# KombiStack quickstart

Get KombiStack running and deploy your first infrastructure stack in under 5 minutes.

## Prerequisites

<CardGroup cols={3}>
  <Card title="Docker" icon="docker">
    Docker Engine 24.0+
  </Card>
  <Card title="System resources" icon="microchip">
    2GB RAM, 5GB disk
  </Card>
  <Card title="SSH access" icon="key">
    SSH key for target servers
  </Card>
</CardGroup>

## Installation

<Tabs>
  <Tab title="Docker Compose (Recommended)">
    ```bash
    # Clone the repository
    git clone https://github.com/KombiverseLabs/KombiStack
    cd KombiStack
    
    # Start services
    docker compose up -d
    
    # Check status
    docker compose ps
    ```
    
    **Services started:**
    - KombiStack Core (API + PocketBase) on port 5260
    - KombiStack UI (SvelteKit) on port 5261
  </Tab>

  <Tab title="Docker (Core only)">
    ```bash
    docker run -d \
      --name kombistack \
      -p 5260:5260 \
      -p 5263:5263 \
      -v kombistack-data:/app/pb_data \
      ghcr.io/kombiverselabs/kombistack:latest
    ```
  </Tab>

  <Tab title="Binary (Linux/macOS)">
    ```bash
    # Download latest release
    curl -fsSL https://raw.githubusercontent.com/KombiverseLabs/KombiStack/main/install.sh | sh
    
    # Start the server
    ./kombistack serve
    ```
  </Tab>

  <Tab title="From source">
    ```bash
    # Prerequisites: Go 1.24+, Bun 1.1+
    git clone https://github.com/KombiverseLabs/KombiStack
    cd KombiStack
    
    # Build backend
    go build -o kombistack ./cmd/kombistack
    
    # Build frontend
    cd app && bun install && bun run build
    
    # Start services
    ./kombistack serve &
    cd app && node build
    ```
  </Tab>
</Tabs>

## First-time setup

<Steps>
  <Step title="Access the dashboard">
    Open [http://localhost:5261](http://localhost:5261) in your browser.
  </Step>

  <Step title="Create admin account">
    On first launch, create your admin account:
    
    - Email: `admin@kombistack.local`
    - Password: Choose a strong password
  </Step>

  <Step title="Configure SSH wallet">
    KombiStack needs SSH keys to access your servers:
    
    1. Navigate to **Wallet** in the sidebar
    2. Click **Generate SSH Key Pair**
    3. Download the public key
    4. Add it to your server's `~/.ssh/authorized_keys`
  </Step>
</Steps>

## Deploy your first stack

<Steps>
  <Step title="Start the wizard">
    Click **New Stack** in the dashboard.
    
    Choose your path:
    - **Easy Wizard:** Answer questions, auto-select StackKit
    - **Expert Wizard:** Full control, choose StackKit manually
  </Step>

  <Step title="Configure infrastructure">
    **Easy Wizard example:**
    
    1. **What do you want to deploy?**
       - Select: "Web applications and databases"
    
    2. **How many servers?**
       - Enter: 1
    
    3. **Server details:**
       - IP: `192.168.1.100`
       - SSH user: `admin`
    
    4. **Which services?**
       - Select: Traefik, Dokploy
  </Step>

  <Step title="Review configuration">
    The wizard generates a `kombination.yaml`:
    
    ```yaml
    version: "1.0"
    name: my-first-stack
    kit: base-homelab
    
    nodes:
      - name: homelab-server
        type: main
        provider: local
        ssh:
          host: 192.168.1.100
          user: admin
    
    services:
      - name: traefik
        type: reverse-proxy
      - name: dokploy
        type: deployment-platform
    ```
  </Step>

  <Step title="Run pre-checks">
    KombiStack validates your configuration:
    
    - ✅ SSH connectivity
    - ✅ Required packages (Docker, etc.)
    - ✅ Disk space
    - ✅ Network configuration
  </Step>

  <Step title="Deploy">
    Click **Deploy** to provision your infrastructure.
    
    **What happens:**
    1. Unifier validates and resolves configuration
    2. OpenTofu generates infrastructure code
    3. `tofu init && tofu plan && tofu apply`
    4. Agents deploy services via Docker Compose
    5. Health checks verify deployment
  </Step>

  <Step title="Monitor progress">
    Watch real-time logs in the **Jobs** tab:
    
    ```
    [INFO] Validating configuration...
    [INFO] Resolving StackKit: base-homelab
    [INFO] Generating OpenTofu configuration...
    [INFO] Running: tofu init
    [INFO] Running: tofu plan
    [INFO] Running: tofu apply
    [INFO] Deploying services...
    [SUCCESS] Stack deployed successfully!
    ```
  </Step>
</Steps>

After deployment, access your services:

| Service | URL | Credentials |
|---------|-----|-------------|
| **Traefik Dashboard** | http://192.168.1.100:8080 | Auto-configured |
| **Dokploy** | http://192.168.1.100:3000 | Check deployment logs |
| **PocketBase Admin** | http://localhost:5260/_/ | Your admin account |

## Common tasks

### Update a stack

```bash
# Edit your kombination.yaml
nano kombination.yaml

# Apply changes
curl -X PUT http://localhost:5260/api/v1/stacks/{id} \
  -H "Content-Type: application/json" \
  -d @kombination.yaml
```

### Add a service

```yaml
services:
  - name: portainer
    type: container-management
    config:
      admin_password: ${PORTAINER_PASSWORD}
```

### Scale nodes

```yaml
nodes:
  - name: worker-1
    type: worker
    ssh:
      host: 192.168.1.101
      user: admin
```

### Destroy a stack

<Warning>
  This will destroy all infrastructure managed by the stack.
</Warning>

```bash
curl -X DELETE http://localhost:5260/api/v1/stacks/{id}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="SSH connection failed">
    **Symptoms:** Pre-checks fail with "SSH connection refused"
    
    **Solutions:**
    1. Verify SSH key is added to server's `authorized_keys`
    2. Check firewall allows SSH (port 22)
    3. Test manually: `ssh admin@192.168.1.100`
    4. Ensure SSH service is running on target server
  </Accordion>

  <Accordion title="OpenTofu errors">
    **Symptoms:** Deployment fails during `tofu apply`
    
    **Solutions:**
    1. Check logs in Jobs tab for specific error
    2. Verify Docker is installed on target server
    3. Ensure sufficient disk space
    4. Check OpenTofu state: `docker exec kombistack ls /app/pb_data/stacks/{id}/`
  </Accordion>

  <Accordion title="Services not starting">
    **Symptoms:** Deployment succeeds but services don't start
    
    **Solutions:**
    1. SSH to server and check Docker: `docker ps -a`
    2. View service logs: `docker logs traefik`
    3. Check port conflicts: `netstat -tulpn | grep :80`
    4. Verify service configuration in PocketBase Admin
  </Accordion>

  <Accordion title="PocketBase admin access">
    **Symptoms:** Cannot access PocketBase Admin UI
    
    **Solutions:**
    1. Ensure Core is running: `curl http://localhost:5260/api/health`
    2. Access admin at: `http://localhost:5260/_/`
    3. Use credentials created during first-time setup
    4. Reset password if needed: `docker exec kombistack ./kombistack admin create`
  </Accordion>
</AccordionGroup>
