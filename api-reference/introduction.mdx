The kombify API provides programmatic access to all kombify tools: Stack, Sim, StackKits, and Cloud.

## Base URLs

| Service | Base URL | Purpose |
|---------|----------|---------|
| **kombify API Gateway** | `https://api.kombify.io/v1` | Central API entry point (Kong) |
| **kombify Stack** | `http://localhost:5260/api/v1` or `https://stack.kombify.io` | Infrastructure orchestration |
| **kombify Sim** | `http://localhost:5270/api/v1` or `https://sim.kombify.io` | Simulation management |
| **kombify Cloud** | Via API Gateway | SaaS platform features |

## Authentication

### Bearer token

All API requests require authentication via Bearer token:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://api.kombify Cloud.io/v1/stacks
```

## Rate limits

| Plan | Requests/minute | Burst |
|------|----------------|-------|
| **Free** | 60 | 100 |
| **Pro** | 300 | 500 |
| **Enterprise** | Unlimited | Unlimited |

Rate limit headers:
```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 59
X-RateLimit-Reset: 1640000000
```

## Response format

### Success response

```json
{
  "success": true,
  "data": {
    "id": "stack_abc123",
    "name": "my-homelab",
    "status": "running"
  }
}
```

### Error response

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid configuration",
    "details": {
      "field": "services.traefik",
      "issue": "Missing required field: domain"
    }
  }
}
```

## Error codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `UNAUTHORIZED` | 401 | Invalid or missing token |
| `FORBIDDEN` | 403 | Insufficient permissions |
| `NOT_FOUND` | 404 | Resource not found |
| `VALIDATION_ERROR` | 400 | Invalid request data |
| `RATE_LIMIT_EXCEEDED` | 429 | Too many requests |
| `INTERNAL_ERROR` | 500 | Server error |

## Pagination

List endpoints support pagination:

```bash
curl "https://api.kombify Cloud.io/v1/stacks?page=1&perPage=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response includes pagination metadata:

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "perPage": 20,
    "totalPages": 5,
    "totalItems": 100
  }
}
```

## Filtering

Filter results with query parameters:

```bash
# Filter by status
curl "https://api.kombify Cloud.io/v1/stacks?status=running"

# Filter by multiple fields
curl "https://api.kombify Cloud.io/v1/stacks?status=running&stackkit=base-homelab"
```

## Sorting

Sort results with the `sort` parameter:

```bash
# Sort by created date (descending)
curl "https://api.kombify Cloud.io/v1/stacks?sort=-created"

# Sort by name (ascending)
curl "https://api.kombify Cloud.io/v1/stacks?sort=name"
```

## Webhooks

Subscribe to events via webhooks:

```bash
curl -X POST https://api.kombify Cloud.io/v1/webhooks \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-app.com/webhooks",
    "events": ["stack.created", "stack.updated", "stack.deleted"]
  }'
```

Available events:
- `stack.created`
- `stack.updated`
- `stack.deleted`
- `stack.deployed`
- `simulation.created`
- `simulation.deleted`

## Quick example

```javascript
import { KombifyClient } from '@kombify/sdk';

const client = new KombifyClient({
  token: process.env.KOMBIFY_TOKEN,
  baseUrl: 'https://api.kombify Cloud.io/v1'
});

// List stacks
const stacks = await client.stacks.list();

// Create a stack
const stack = await client.stacks.create({
  name: 'my-homelab',
  stackkit: 'base-homelab',
  config: {
    domain: 'home.local'
  }
});

// Deploy
await client.stacks.deploy(stack.id);
```
