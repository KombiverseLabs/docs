# What is kombify?

## The kombify family

kombify is a suite of tools that work together to simplify infrastructure management:

<AccordionGroup>
  <Accordion title="kombify Stack - Hybrid control plane" icon="server">
    The orchestration engine that manages your infrastructure. Define your desired state in `kombination.yaml`, and Stack handles validation, provisioning, and deployment.
    
    **Key features:**
    - Spec-driven architecture with CUE validation
    - Multi-node management via gRPC agents
    - OpenTofu integration for IaC
    - Embedded PocketBase for state management
    
    **License:** MIT + AGPL-3.0 (Open Source)
  </Accordion>

  <Accordion title="kombify Sim - Simulation engine" icon="flask">
    Test your infrastructure before deploying to real hardware. Sim creates lightweight Docker containers that simulate your entire homelab.
    
    **Key features:**
    - Docker-based simulation nodes
    - Real SSH access to simulated servers
    - Pre-built templates for common setups
    - Full REST API for automation
    
    **License:** MIT (Open Source)
  </Accordion>

  <Accordion title="kombify StackKits - IaC blueprints" icon="boxes-stacked">
    Pre-validated infrastructure blueprints with CUE schemas. StackKits provide battle-tested configurations for common homelab scenarios.
    
    **Key features:**
    - CUE schema validation
    - Built-in blueprints (base-homelab, ha-homelab, modern-homelab)
    - Extensible and customizable
    - Community-contributed kits
    
    **License:** MIT (Open Source)
  </Accordion>

  <Accordion title="kombify Sphere - SaaS platform" icon="cloud">
    The managed platform that brings everything together. Sphere provides SSO, billing, team management, and a unified dashboard.
    
    **Key features:**
    - Single Sign-On with Zitadel
    - Subscription management with Stripe
    - Tool launcher for Stack, Sim, and more
    - Team collaboration features
    
    **License:** Proprietary (SaaS)
  </Accordion>
</AccordionGroup>

## How it works

```yaml kombination.yaml
# This is all you write
stack:
  name: my-homelab
  domain: home.local

nodes:
  - hostname: proxmox-1
    type: proxmox
    role: hypervisor

services:
  - traefik      # Reverse proxy
  - authelia     # SSO
  - homepage     # Dashboard
  - immich       # Photos
```

**What kombify does:**
1. **Validates** your config with CUE schemas
2. **Simulates** the setup in Docker (optional)
3. **Generates** OpenTofu/Terraform code
4. **Deploys** to your infrastructure
5. **Manages** ongoing state and updates

## Deployment options

<Tabs>
  <Tab title="SaaS (Recommended)">
    Use kombify Sphere for the easiest experience.
    
    ```bash
    # Sign up at app.kombisphere.io
    # Launch tools from the dashboard
    # No infrastructure to manage
    ```
    
    **Best for:**
    - Getting started quickly
    - Teams and collaboration
    - Managed updates and support
    - Enterprise SSO
  </Tab>

  <Tab title="Self-hosted">
    Run kombify Stack on your own infrastructure.
    
    ```bash
    # Docker Compose
    curl -O https://raw.githubusercontent.com/soulcreek/KombiStack/main/docker-compose.yml
    docker compose up -d
    ```
    
    **Best for:**
    - Privacy-conscious users
    - Full control over data
    - No subscription costs
    - Custom modifications
  </Tab>

  <Tab title="Hybrid">
    Self-host the core tools, use Sphere for management.
    
    ```bash
    # Run Stack locally
    docker compose up -d
    
    # Connect to Sphere for SSO and collaboration
    ```
    
    **Best for:**
    - Data sovereignty requirements
    - Hybrid cloud/on-prem setups
    - Gradual migration to SaaS
  </Tab>
</Tabs>

## Tech stack

| Component | Technology | Purpose |
|-----------|------------|---------|
| **Backend** | Go 1.24+ | Core services and agents |
| **Frontend** | SvelteKit 2.x | Modern reactive UI |

| **Validation** | CUE 0.15+ | Type-safe configuration |
| **API gateway** | Kong OSS | Unified API entry point |
| **Auth** | Zitadel | Enterprise OIDC provider |
