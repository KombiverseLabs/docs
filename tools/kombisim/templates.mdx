## Template details

### Homelab basic

**Use case:** Simple web application with database

**Nodes:**
- `web-server` (SSH: 2222) - Web application server
- `database` (SSH: 2223) - Database server

**Apply template:**
```bash
curl -X POST http://localhost:5270/api/v1/templates/homelab-basic/apply \
  -H "Content-Type: application/json" \
  -d '{
    "simulation_name": "my-homelab",
    "auto_start": true
  }'
```

**Example deployment:**
```bash
# SSH to web-server
ssh -p 2222 root@localhost

# Deploy web app
docker run -d --name web -p 80:80 nginx

# SSH to database
ssh -p 2223 root@localhost

# Deploy database
docker run -d --name db -e POSTGRES_PASSWORD=secret postgres
```

### Homelab advanced

**Use case:** Production-grade homelab with full stack

**Nodes:**
- `proxy` (SSH: 2222) - Reverse proxy (Traefik)
- `app-1` (SSH: 2223) - Application server 1
- `app-2` (SSH: 2224) - Application server 2
- `database` (SSH: 2225) - Database server
- `storage` (SSH: 2226) - Storage/NAS server

**Apply template:**
```bash
curl -X POST http://localhost:5270/api/v1/templates/homelab-advanced/apply \
  -d '{"simulation_name": "production-homelab", "auto_start": true}'
```

**Example deployment:**
```bash
# Deploy Traefik on proxy
ssh -p 2222 root@localhost
docker run -d --name traefik -p 80:80 -p 443:443 traefik:3.0

# Deploy apps
ssh -p 2223 root@localhost
docker run -d --name app nginx

# Deploy database
ssh -p 2225 root@localhost
docker run -d --name postgres -e POSTGRES_PASSWORD=secret postgres

# Deploy storage
ssh -p 2226 root@localhost
docker run -d --name nextcloud -p 8080:80 nextcloud
```

## Customizing templates

You can modify templates after applying them:

```bash
# Apply template
curl -X POST http://localhost:5270/api/v1/templates/homelab-basic/apply \
  -d '{"simulation_name": "custom", "auto_start": false}'

# Add additional node
curl -X POST http://localhost:5270/api/v1/simulations/{id}/nodes \
  -d '{"name": "cache-server", "location": "local"}'

# Start simulation
curl -X POST http://localhost:5270/api/v1/simulations/{id}/start
```
