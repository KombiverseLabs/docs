## SSO Configuration

### Supported providers

- Okta
- Azure Active Directory
- Google Workspace
- OneLogin
- Ping Identity
- Any SAML 2.0 / OIDC provider

### OIDC setup

<Steps>
  <Step title="Get Cloud OIDC details">
    In Cloud: **Settings** → **Security** → **SSO**
    
    Copy:
    - Redirect URI: `https://Cloud.kombify.dev/auth/callback`
    - Client ID: (provided)
  </Step>
  <Step title="Configure your IdP">
    In your identity provider:
    - Add new OIDC application
    - Set redirect URI
    - Configure scopes: `openid`, `profile`, `email`
    - Copy Client Secret
  </Step>
  <Step title="Complete Cloud setup">
    Enter in Cloud:
    - Issuer URL
    - Client ID
    - Client Secret
  </Step>
  <Step title="Configure role mapping">
    Map IdP groups to Cloud roles:
    - `Cloud-admins` → Admin
    - `Cloud-operators` → Operator
    - `Cloud-viewers` → Viewer
  </Step>
  <Step title="Enable SSO">
    Toggle **Enable SSO**
    
    Optionally: **Enforce SSO** (disable password login)
  </Step>
</Steps>

### SAML setup

<Accordion title="SAML configuration details">
**Cloud SAML metadata:**
- Entity ID: `https://Cloud.kombify.dev/saml/metadata`
- ACS URL: `https://Cloud.kombify.dev/saml/acs`
- SLO URL: `https://Cloud.kombify.dev/saml/slo`

**Required attributes:**
- `email` (required)
- `firstName` (optional)
- `lastName` (optional)
- `groups` (for role mapping)
</Accordion>

### SCIM provisioning

Automatically sync users from your IdP:

1. Enable SCIM in **Settings** → **Security** → **SCIM**
2. Copy SCIM endpoint and token
3. Configure SCIM in your IdP
4. Map groups to Cloud roles

**Supported operations:**
- Create user
- Update user
- Deactivate user
- Group membership sync

## Self-hosted deployment

Run Cloud in your own infrastructure.

### Deployment options

| Option | Description |
|--------|-------------|
| **Kubernetes** | Helm chart for K8s clusters |
| **Docker Compose** | Single-node deployment |
| **VM/Bare metal** | Traditional server deployment |

### Requirements

<Tabs>
  <Tab title="Kubernetes">
    **Minimum:**
    - Kubernetes 1.28+
    - 3 nodes (4 CPU, 8 GB RAM each)
    - 100 GB storage (SSD recommended)
    - Ingress controller
    - cert-manager (for TLS)
    
    **Recommended:**
    - 5+ nodes for HA
    - Managed PostgreSQL
    - Managed Redis
  </Tab>
  <Tab title="Docker Compose">
    **Minimum:**
    - 4 CPU cores
    - 16 GB RAM
    - 100 GB SSD storage
    - Docker 24+
    - Docker Compose v2
    
    **Note:** Single-node deployment, not HA
  </Tab>
</Tabs>

### Installation

<Steps>
  <Step title="Get license">
    Contact sales for Enterprise license key
  </Step>
  <Step title="Download package">
    ```bash
    curl -L https://get.kombify.dev/Cloud-enterprise | bash
    ```
  </Step>
  <Step title="Configure">
    Edit `Cloud-config.yaml`:
    ```yaml
    license_key: "your-license-key"
    domain: "Cloud.yourcompany.com"
    database:
      host: "postgres.internal"
      name: "Cloud"
    redis:
      host: "redis.internal"
    ```
  </Step>
  <Step title="Deploy">
    **Kubernetes:**
    ```bash
    helm install Cloud ./Cloud-chart -f Cloud-config.yaml
    ```
    
    **Docker Compose:**
    ```bash
    docker compose up -d
    ```
  </Step>
  <Step title="Initialize">
    Access `https://Cloud.yourcompany.com/setup` to complete initial configuration.
  </Step>
</Steps>

### Updates

Self-hosted instances receive updates via:
- **Automatic** — Enabled by default, applies during maintenance window
- **Manual** — Download and apply updates yourself

```bash
# Check for updates
Cloud-ctl update check

# Apply update
Cloud-ctl update apply
```
